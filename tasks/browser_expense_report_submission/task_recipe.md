# Task Recipe: Browser Expense Report Submission

Use a browser-based expense system to enter items, attach receipts, and stop before final submission when needed.

## Best Used When

- A user files monthly expenses in a web portal
- Receipts must be attached and categorized
- A draft needs to be prepared for later review

## Inputs

- Expense portal name
- Date range or report name
- Receipt folder
- Draft-only or submit-now instruction

## Success Conditions

- The correct portal and report page were opened
- Expenses and receipts were added to the right fields
- The report stops at draft or final submit as requested

## Prompt Examples

- `Edgeで経費精算システムを開いて、今月の交通費を入力して下書き保存して。`
- `ブラウザで経費申請を作って、領収書フォルダのPDFを添付して。最終送信はしないで。`

## Tips

- Say whether login is already complete or may need manual intervention
- Name the receipt folder or downloads location
- Final submission is externally visible, so state it only when truly intended
