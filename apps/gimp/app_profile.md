# GIMP App Profile

## Purpose

Raster image editing with dense toolboxes, dialogs, and modal state changes.

## Main Screens

- Main canvas window
- Toolbox and dock panels
- Export and save dialogs

## Typical Tasks

- Open image
- Crop or resize
- Export in another format
- Draw simple illustrations or annotations

## UI Structure

- Menu bar
- Toolbox
- Canvas
- Layer and properties docks

## Important Menus

- File
- Edit
- Image
- Layer
- Colors

## Important Shortcuts

- `Ctrl+O`
- `Ctrl+S`
- `Ctrl+Shift+E`
- `Shift+C`
- `P`
- `B`

## Common Pitfalls

- Tool mode changes after dialog interaction
- Export and save are distinct
- Unsaved changes prompts appear frequently

## Version Differences

- Dock layout and iconography vary by version and theme

## Operational Notes

- Confirm GIMP has fully launched before selecting tools
- Confirm active document or blank canvas before destructive edits
- Prefer keyboard shortcuts for tool selection when icon layouts are ambiguous
- Draw one visible stroke at a time, then re-observe the canvas
- Handle overwrite prompts according to the requested output path and filename

## Common Success Patterns

- Use menu shortcuts when tools are visually ambiguous
- Reconfirm focused canvas before drawing or transforming
- After opening from Windows Search, wait for toolbox and canvas to appear before typing or dragging
